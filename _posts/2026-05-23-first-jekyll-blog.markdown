## 为什么要搭建独立博客？

话说最近老师布置了一个作业，让我们搭建个人博客。一开始我是拒绝的 —— 都2026年了，谁还看独立博客啊，发发朋友圈不香吗？

但真正动手做下来，发现 Jekyll 博客真的很有意思！今天就来聊聊我的第一次体验~

### Jekyll 是什么？

简单来说，Jekyll 是一个**静态网站生成器**。你可以用 Markdown 写文章，它会自动帮你转换成漂亮的网页。

> 它不需要数据库，不需要复杂的配置，特别适合个人博客和技术文档网站。

### 搭建过程

**第一步：安装环境**

我用的是 Windows 系统，安装过程稍微有点曲折（老师说要装 Ruby，我差点以为自己在挖矿 😂）：

```bash
# 安装 Bundler
gem install bundler

# 创建新站点
jekyll new my-blog

# 进入目录
cd my-blog

# 启动服务器
bundle exec jekyll serve
```

**第二步：目录结构**

Jekyll 的项目结构很清晰：

```
my-blog/
├── _posts/          # 博客文章放这里
├── _layouts/        # 页面布局模板
├── _includes/       # 可复用的组件
├── _config.yml      # 配置文件
├── assets/          # CSS、JS、图片
└── index.md        # 首页
```

**第三步：写文章**

这才是重点！写文章超级简单：

```markdown
---
layout: post
title: "我的第一篇文章"
date: 2026-05-23
categories: 生活
---

这里是正文内容，用 Markdown 格式书写。

## 小标题

- 列表项1
- 列表项2

就这么简单！
```

### 主题选择

Jekyll 有很多免费主题，我选了一个简洁的 Minima 主题。后来发现可以自己定制 CSS，就动手改了一版，现在看起来还挺像样的！

### GitHub Pages 免费托管

最香的是可以免费托管到 GitHub Pages！只要把代码推送到 GitHub，就能在网上访问你的博客了，而且完全免费！

### 优点 vs 缺点

**优点：**
- ✅ 简单易用，写作体验好
- ✅ 加载速度快，纯静态页面
- ✅ 免费托管，省钱
- ✅ 可以深度定制
- ✅ Git 版本管理，再也不怕文章丢失

**缺点：**
- ❌ 评论功能需要第三方支持（ Disqus 等）
- ❌ 动态功能受限（比如搜索、点赞等）
- ❌ Windows 下安装有点麻烦

### 心得体会

虽然搭建过程遇到了一些坑（比如 Ruby 版本问题、bundle install 超时等），但最终看到自己的博客上线，真的超有成就感！

现在我可以在任何文本编辑器里写文章，然后 `git push` 就发布了，这种感觉真的很棒！而且因为是纯静态页面，加载速度也很快。
