## 什么是组合式 API

Vue 3 引入了组合式 API（Composition API），这是一种全新的代码组织方式，相比传统的选项式 API，它提供了更好的代码复用和逻辑组织能力。

### 为什么需要组合式 API

在 Vue 2 中，我们使用选项式 API 来组织代码：

```javascript
export default {
  data() {
    return {
      count: 0
    }
  },
  methods: {
    increment() {
      this.count++
    }
  }
}
```

这种方式在处理复杂组件时会遇到问题，相关的逻辑分散在不同的选项中，难以维护。

### 组合式 API 的基本用法

```javascript
import { ref, onMounted } from 'vue'

export default {
  setup() {
    const count = ref(0)
    
    const increment = () => {
      count.value++
    }
    
    onMounted(() => {
      console.log('Component mounted')
    })
    
    return {
      count,
      increment
    }
  }
}
```

### ref 和 reactive

`ref` 用于创建响应式的基本类型数据，而 `reactive` 用于创建响应式对象：

```javascript
import { ref, reactive } from 'vue'

const count = ref(0)
const state = reactive({
  name: 'Vue 3',
  version: 3
})
```

### 计算属性

使用 `computed` 创建计算属性：

```javascript
import { ref, computed } from 'vue'

const firstName = ref('John')
const lastName = ref('Doe')

const fullName = computed(() => {
  return `${firstName.value} ${lastName.value}`
})
```

### 生命周期钩子

组合式 API 提供了对应的生命周期钩子：

- `onMounted` - 组件挂载后
- `onUpdated` - 组件更新后
- `onUnmounted` - 组件卸载前

### 总结

组合式 API 为 Vue 3 带来了更灵活的代码组织方式，特别适合处理复杂的业务逻辑。通过将相关逻辑组合在一起，可以提高代码的可读性和可维护性。

> 学习 Vue 3 的组合式 API 是前端开发者的必备技能之一。
